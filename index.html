Emacs 使用感受

2020-12-10 10:23
suzumiya natsuhi

不知不觉使用 emacs 已经有两年了。其实一开始看 Emacs 那篇自带的极短的介绍要更早一点，现在看当时的那种喜悦的感觉（用上了众多名人都推荐的软件）更多的是 naive 和开心。

后来从开始使用无配置的 emacs ，然后用 Spacemacs 的时候有眼前有一亮的感觉。然后逐渐学习使用 org-mode 并且针对 org-mode 进行自己的配置，也收获了很多开心的时光。

慢慢的又转向使用 doom-emacs 感觉又有眼前一亮的感觉，主要是启动速度更快了，然后基本上开箱即用。然后也学到一些自己配置的方法，虽然 spacemacs 也可以自定义，但是启动一次还是蛮长的。

然后逐渐学到了 org-roam, org-download, org-capture, org-pdftool 等等都是很好用的插件。不过现在已经很少纠结于配置了，现在的已经够用了。

最后的最后，送一句传教语：

Emacs loves me, emacs loves you.

《被讨厌的勇气》和宁静祷文

2020-12-08 05:50
suzumiya natsuhi

  • 被讨厌的勇气

    正如本书的核心之一“课题分离”所建议的，其实“被讨厌的勇气”就是“课题分离”一个非常重要的形式。例如：讨厌是我不能决定的事情，也就是对方的课题，但是有勇气是我的课题。我插手你的课题让你不讨厌我其实是没有意义的（也就是我不能插手别人的课题），同时自己的课题“勇气”需要自己来解决。从这一点上来看，标题和阿德勒核心的指导思想是一致的，只不过用另一种形式来展现而已。

  • 自己的课题和别人的课题

    认识到这两个概念的本身，就会激发人的课题“分离”意识。这就很像软件工程界的 Decoupling 。当确认之后，对于自己的课题，可以毫无保留的全力以赴；但是对于别人的课题则要慎之又慎，不可插手。

  • “分离”的智慧与宁静祷文

    以下是宁静祷文的内容：

    上帝，请赐予我接受不能改变之事的平静， 和我可以改变事情的勇气， 以及区分这两者的智慧。

    智慧就是区分本身，拥有智慧的人同时会拥有内心的宁静和改变的勇气。
